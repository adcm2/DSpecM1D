Documentation
Apply Doxygen comments consistently — t1.cpp now has excellent Doxygen coverage but spectra_master.h has none. The public API (all MAT_*, RV_*, CalculateForce_* methods) should at minimum have a @brief, @param, and @return.

Add a Doxyfile to the repository root so documentation can be generated with a single doxygen invocation.

Testing
There are no tests — at minimum, add a tests/ directory with a few CTest cases that verify:
The LtG maps (LtG_S, LtG_T, LtG_R) return expected indices for known inputs.
The assembled matrices are symmetric.
A known analytic mode frequency is recovered to within tolerance.
Performance
The CalculateForce_* methods re-construct the Wigner matrix on every call for the same source. If the source location is fixed (which it is within a single run), cache the Wigner matrix as a member or pass it in.

Consider using Eigen::SparseMatrix<Complex> directly for the frequency-domain solve rather than assembling a real matrix and promoting it at solve time — this avoids an extra copy per frequency.
