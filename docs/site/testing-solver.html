Filtering, Solver API, And SEM Tests

These tests cover the preferred public solver path, the guarded filter wrapper, and a narrow set of low-level SEM contracts that fail close to the bug source.

Filtering

  • FilterOptions defaults are checked directly.
  • Both applyFilter(...) overloads are checked for valid execution on synthetic inputs.
  • Invalid matrix widths are rejected explicitly at the wrapper boundary.

Preferred And Legacy Solver Entry Points

  • The preferred SparseFSpec::spectra(InputParametersNew &) path is covered directly.
  • The SEM-reuse overloads are checked for consistent output shape and agreement.
  • The legacy single-SEM and multi-SEM overloads now have small direct regression tests on a tiny setup.

SEM Contracts

  • Local-to-global indexing invariants are checked on a minimal setup.
  • Receiver/source element lookup is checked for in-mesh bounds.
  • Main SEM matrices are checked for consistent square shapes and structural non-emptiness.
  • Selected receiver-vector helpers are checked for expected dimensions and non-zero support.
