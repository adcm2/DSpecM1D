Parameter Files

DSpecM1D currently reads an ordered line-based input format. It does not read keyed name value pairs.

Parsing Rules

  • Blank lines are ignored.
  • Comment lines are ignored.
  • Quoted and unquoted scalar values are accepted where the parser expects them.
  • After comments and blanks are skipped, the remaining values must appear in the exact expected order.

Recommended API

For release-facing code, prefer InputParametersNew. It preserves the legacy file format while aggregating the parsed parameters, Earth model, source, receiver geometry, and frequency axis in one object.

Why The Docs Call This Out Explicitly

Earlier README-style examples could be read as if the input format were keyed. The release docs now describe the real parser behavior directly so users do not prepare the wrong kind of input file.
