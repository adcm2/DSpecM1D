Release Notes

The first public DSpecM1D release separates protected scientific-reproduction workflows from day-to-day library testing, while adding a public seismogram-generation CLI, documentation, and downstream packaging support.

Highlights

  • Added a modular self-contained unit/component test suite.
  • Added the public generate_seismogram executable for parameter-file-driven time-domain output.
  • Kept t1 as the tutorial and smoke example.
  • Preserved ex1 through ex7 as paper-reproduction workflows.
  • Added Doxygen-driven API documentation and a static documentation website.
  • Added install rules and downstream find_package(DSpecM1D) support.
  • Added a local site preview target for the documentation website.

Release Gates

  • Build the library, tutorial, and examples.
  • Run modular unit/component tests.
  • Run the public generate_seismogram smoke check.
  • Run the self-contained t1 smoke test.
  • Run protected paper-reproduction workflows in the supported environment.
  • Build the website and API docs.

What Comes Next

  • Highest priority: further cleanup and small API improvements.
  • Next priority: Love numbers.
  • Next priority: optional LAPACK banded-solver support.
