Smoke And Integration Checks

These checks give a quick end-to-end signal that the build tree is sane before you spend time on heavier validation.

Covered Checks

  • smoke_generate_seismogram: runs the public CLI on a small bundled parameter file and verifies that a non-empty time-series output file is written.
  • smoke_build_benchmark_ex1: verifies that the protected ex1 paper example still builds.
  • smoke_tutorial_t1: runs the self-contained tutorial/integration workflow.

Guarded Runtime

The smoke checks are configured with capped OpenMP thread counts, and the heavier runtime paths also use timeouts and serial execution, so routine validation does not consume an uncontrolled number of CPU cores. For manual CLI, tutorial, or benchmark runs, set OMP_NUM_THREADS explicitly.

Recommended Use

ctest --test-dir build/dev --output-on-failure -L dspecm1d_smoke

Run these after the unit/component suites and before the paper-reproduction workflows; if they fail, there is usually no point running the heavier examples yet. Use a clean build tree when reporting failures so stale dependency tests are not mixed into the result.
