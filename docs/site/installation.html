Installation

Build from source, install the package, and use the generated CMake config in downstream projects.

Requirements

DSpecM1D requires a C++23 compiler, CMake 3.24 or newer, FFTW3, a BLAS/LAPACK implementation, and OpenMP. The remaining dependencies are fetched automatically from pinned revisions during the first configure.

sudo apt install libfftw3-dev libopenblas-dev liblapack-dev ninja-build

cmake -S . -B build/release -G Ninja
cmake --build build/release --parallel

Quick Start

The recommended first executable is t1. It is the self-contained tutorial and the default runtime smoke example because it does not depend on the external comparison datasets used by the protected paper workflows.

cmake -S . -B build/dev -G Ninja \
  -DDSPECM1D_BUILD_TUTORIALS=ON
cmake --build build/dev --target t1
./build/dev/bin/t1

Installed Package

The release installs headers plus a package config for downstream CMake use. A consumer can locate the package and link to the public interface target:

find_package(DSpecM1D REQUIRED)
target_link_libraries(my_target PRIVATE DSpecM1D::DSpecM1D)

Documentation Build And Local Preview

The static site and Doxygen output are integrated into the CMake build. Use website to generate the site and preview-website to serve it locally.

cmake -S . -B build/docs -G Ninja -DDSPECM1D_BUILD_DOCS=ON
cmake --build build/docs --target website
cmake --build build/docs --target preview-website

The preview server runs at http://127.0.0.1:8000 until you stop it.

Downstream Reproducibility Note

The installed package recreates pinned dependency targets with FetchContent. That keeps the package usable without exporting third-party targets directly, but a first downstream configure may need network access unless those dependencies are already cached locally.
