Installation

Build from source, run the public CLI, install the package, and use the generated CMake config in downstream projects.

Requirements

DSpecM1D requires a C++23 compiler, CMake 3.24 or newer, FFTW3, and an OpenMP-capable compiler toolchain. The remaining third-party C++ dependencies are fetched automatically from pinned revisions during the first configure. NetCDF, BLAS, and LAPACK are not required for the first public release build.

sudo apt install build-essential cmake git libfftw3-dev

cmake -S . -B build/release
cmake --build build/release --parallel

Ninja is optional. If it is installed and you prefer it, add -G Ninja to the configure command.

If you switch CMake generators for an existing build tree, use a new build directory or remove the old one first. CMake build trees are generator-specific.

Quick Start

The recommended first executable is generate_seismogram. It reads a standard parameter file and writes a filtered time-domain seismogram to the output prefix specified in that file.

cmake -S . -B build/dev \
  -DDSPECM1D_BUILD_TOOLS=ON
cmake --build build/dev --target generate_seismogram
cd build/dev
export OMP_NUM_THREADS=2
./bin/generate_seismogram data/params/t1.txt

For the bundled t1.txt file, this writes output/t1.out_t.out inside the active build tree. More generally, the destination comes from the parameter-file output_prefix; the CLI does not take a separate output-path override in this first release.

The bundled t1 executable remains the self-contained tutorial and guarded runtime smoke example. The paper-reproduction examples ex1 through ex7 remain separate on purpose.

Installed Package

The release installs headers plus a package config for downstream CMake use. A consumer can locate the package and link to the public interface target:

find_package(DSpecM1D REQUIRED)
target_link_libraries(my_target PRIVATE DSpecM1D::DSpecM1D)

Documentation Build And Local Preview

The static site and Doxygen output are integrated into the CMake build. Use website to generate the site and preview-website to serve it locally.

cmake -S . -B build/docs -DDSPECM1D_BUILD_DOCS=ON
cmake --build build/docs --target website
cmake --build build/docs --target preview-website

The preview server runs at http://127.0.0.1:8000 until you stop it.

Downstream Reproducibility Note

The installed package recreates the remaining pinned third-party targets with FetchContent. That keeps the package usable without exporting third-party targets directly, but a first downstream configure may need network access unless those dependencies are already cached locally.
