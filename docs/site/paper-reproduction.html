Paper Reproduction Examples

The executables ex1 through ex7 are protected workflows for reproducing the results discussed in the accompanying paper.

Important: these examples intentionally keep their external comparisons. They are not accidental couplings and should not be simplified into generic unit tests.

Role In The Release

  • generate_seismogram is the public end-user CLI, while t1 remains the self-contained tutorial and smoke example.
  • ex1 through ex7 remain paper-reproduction validation workflows.
  • The modular test suite is the main automated regression safety net for code changes.

External Comparison Inputs

The protected examples compare against outputs from other tools such as YSpec, SpecNM, and MinEOS. The canonical comparison files used by these workflows are now stored in the repository under data/reference/ and copied into the build tree with the rest of the project data.

The repository folder plotting/ is also part of this protected workflow: it contains the example plotting scripts as well as the output locations used by the paper-oriented benchmark runs. For this release it should be treated as part of the scientific validation layout, not as a generic optional plotting add-on.

Validation Expectation

These examples should continue to build and run with the same output flow and comparison logic as the current known-good baseline. They are best used as a heavier validation layer after unit/component tests plus the generate_seismogram and t1 smoke checks have already passed.

Recommended Execution Order

  1. Configure and build the library.
  2. Run the modular unit/component tests.
  3. Run the self-contained t1 smoke example.
  4. Run the protected paper-reproduction workflows in the supported environment.
