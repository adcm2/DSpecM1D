Library Testing

The main regression safety net is a suite of small, synthetic, local tests, with separate linked pages for each major test area and the heavier paper-reproduction workflows kept apart on purpose.

Testing Pages

  • Smoke and integration checks
  • Parser and workflow-context tests
  • Reference I/O and output-writer tests
  • Filtering, solver API, and SEM/component tests
  • Coverage status and residual risks

Test Categories

  • Unit: parser rules, quoted and unquoted values, range checking, helper functions, normalization, and file-format readers.
  • Component: output writers, filter option validation, and minimal SEM/local-to-global checks.
  • Smoke: runtime execution of t1.
  • Migration: temporary tolerance-based checks used during dependency in-housing work, such as the ex2 trace comparison.
  • Paper validation: environment-specific workflows for ex1 through ex7.

Recommended Fast Developer Check

cmake -S . -B build/dev \
  -DDSPECM1D_BUILD_TESTS=ON \
  -DDSPECM1D_BUILD_TUTORIALS=ON \
  -DDSPECM1D_BUILD_BENCHMARKS=ON \
  -DDSPECM1D_ENABLE_SMOKE_TESTS=ON
cmake --build build/dev --parallel
ctest --test-dir build/dev --output-on-failure -L "unit|component"
ctest --test-dir build/dev --output-on-failure -L dspecm1d_smoke

Migration-Phase Check

A temporary migration-only check is available during the current dependency cleanup work. It reruns ex2 and compares the DSpecM1D traces against the checked-in reference using numerical tolerances rather than exact file identity.

cmake -S . -B build/dev \
  -DDSPECM1D_BUILD_TESTS=ON \
  -DDSPECM1D_BUILD_BENCHMARKS=ON \
  -DDSPECM1D_ENABLE_MIGRATION_TESTS=ON
ctest --test-dir build/dev --output-on-failure -L migration

Smoke-Test Safety

The t1 smoke test is configured to run serially, with a capped OpenMP thread count and a timeout, so routine validation does not consume an uncontrolled number of CPU cores. For manual tutorial or benchmark runs, set OMP_NUM_THREADS explicitly.
