Library Testing

The main regression safety net is a suite of small, synthetic, local tests. Full waveform comparisons are reserved for protected reproduction workflows, not for everyday debugging.

Test Categories

  • Unit: parser rules, quoted and unquoted values, range checking, helper functions, normalization, and file-format readers.
  • Component: output writers, filter option validation, and minimal SEM/local-to-global checks.
  • Smoke: runtime execution of t1.
  • Paper validation: environment-specific workflows for ex1 through ex7.

Recommended Fast Developer Check

cmake -S . -B build/dev -G Ninja \
  -DDSPECM1D_BUILD_TESTS=ON \
  -DDSPECM1D_BUILD_TUTORIALS=ON \
  -DDSPECM1D_BUILD_BENCHMARKS=ON \
  -DDSPECM1D_ENABLE_SMOKE_TESTS=ON
cmake --build build/dev --parallel
ctest --test-dir build/dev --output-on-failure -L "unit|component"
ctest --test-dir build/dev --output-on-failure -L smoke

Smoke-Test Safety

The t1 smoke test is configured to run serially, with a capped OpenMP thread count and a timeout, so routine validation does not consume an uncontrolled number of CPU cores.

What The Small Tests Cover

The self-contained suite now targets parsing behavior, InputParametersNew, normalization-factor selection, FilterOptions, filter validation, writer truncation and bounds behavior, reference-series readers, helper logic, and minimal SEM mapping invariants.
