Testing Coverage

This page folds the API coverage matrix into the docs site so the current test surface and the intentional remaining gaps are visible without leaving the website.

Direct Modular Coverage

  • InputParametersNew, SpectraRunContext, preferred SparseFSpec overloads, and SEM(InputParametersNew).
  • FilterOptions and both applyFilter(...) overloads.
  • Reference-data loaders and output-writer overloads.
  • Selected helper utilities and targeted SEM matrix/vector/indexing contracts.

Paper-Example And Smoke Coverage

  • t1 remains the self-contained runtime smoke check.
  • ex1 is still the clearest modern public-path example.
  • ex2 through ex7 remain the main protection for scientific workflow fidelity and the specialist legacy/low-level paths used in the paper.

Residual Risks

  • Some low-level SEM helpers are still covered mainly through solver execution and paper examples rather than one-function unit tests.
  • Legacy SparseFSpec workflows are now directly sampled by small tests, but the paper examples remain the authoritative scientific check for them.
  • Specialist low-level access such as meshModel() is lightly contract-checked and still mainly relevant to workflows like ex6.
